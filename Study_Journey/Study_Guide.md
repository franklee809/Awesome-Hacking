# Awesome Hacking Study Guide

This guide helps you navigate the [Awesome-Hacking](https://github.com/franklee809/Awesome-Hacking) repository. The repository is a "meta-list," meaning it contains links to other curated lists of hacking resources.

## 🎓 Recommended Learning Path

If you are new to hacking, the sheer number of lists can be overwhelming. Here is a recommended order of study:

### 1. Foundations (Start Here)
Before diving into specific exploits, build a strong foundation.
- **[Hacking Resources](https://github.com/vitalysim/Awesome-Hacking-Resources)**: General collection to get started.
- **[Cyber Skills](https://github.com/joe-shenouda/awesome-cyber-skills)**: Environments to practice partially legally and safely.
- **[Hacker Roadmap](https://github.com/sundowndev/hacker-roadmap)**: A guide for amateur pen testers.

### 2. Core Methodologies
Learn the standard processes used in security assessments.
- **[Pentest](https://github.com/enaqx/awesome-pentest)**: The definitive list for penetration testing tools and resources.
- **[Bug Bounty](https://github.com/djadmin/awesome-bug-bounty)**: Learn how to hunt for bugs in real-world programs.
- **[AppSec](https://github.com/paragonie/awesome-appsec)**: Understanding application security is crucial for breaking it.

### 3. Domain Specifics (Pick Your Path)
Once you know the basics, specialize in one or more areas.

**Web & Network**
- [Web Hacking](https://github.com/infoslack/awesome-web-hacking)
- [API Security Checklist](https://github.com/shieldfy/API-Security-Checklist)
- [Pcaptools](https://github.com/caesar0301/awesome-pcaptools) (Network Analysis)

**Mobile & Hardware**
- [Android Security](https://github.com/ashishb/android-security-awesome)
- [OSX and iOS Security](https://github.com/ashishb/osx-and-ios-security-awesome)
- [Embedded and IoT Security](https://github.com/fkie-cad/awesome-embedded-and-iot-security)
- [Vehicle Security](https://github.com/jaredthecoder/awesome-vehicle-security)

**Systems & Binary Exploitation**
- [Exploit Development](https://github.com/FabioBaroni/awesome-exploit-development)
- [Reversing](https://github.com/fdivrp/awesome-reversing) (Reverse Engineering)
- [Malware Analysis](https://github.com/rshipp/awesome-malware-analysis)
- [Windows Exploitation](https://github.com/yeyintminthuhtut/Awesome-Advanced-Windows-Exploitation-References)

### 4. Advanced & Niche
- [Red Teaming](https://github.com/yeyintminthuhtut/Awesome-Red-Teaming): Simulating full-scope adversarial attacks.
- [DevSecOps](https://github.com/devsecops/awesome-devsecops): Integrating security into DevOps.
- [Industrial Control System Security](https://github.com/hslatman/awesome-industrial-control-system-security): Critical infrastructure security.
- [Adversarial Machine Learning](https://github.com/yenchenlin/awesome-adversarial-machine-learning): Hacking AI models.

## 🛠️ Essential Toolkits
Every hacker needs a toolkit.
- **[SecLists](https://github.com/danielmiessler/SecLists)**: Usernames, passwords, URLs, patterns, etc. (Must have!)
- **[PayloadsAllTheThings](https://github.com/swisskyrepo/PayloadsAllTheThings)**: Useful payloads for bypassing security.
- **[GTFOBins](https://gtfobins.github.io)**: Unix binaries that can be exploited.
- **[CyberChef](https://gchq.github.io/CyberChef/)**: The "Cyber Swiss Army Knife" for decoding/encoding.

## 📚 Continuous Learning
Stay updated and practice.
- **[CTF](https://github.com/apsdehal/awesome-ctf)**: Capture The Flag competitions to practice skills.
- **[Sec Talks](https://github.com/PaulSec/awesome-sec-talks)**: Watch conference talks to learn new techniques.
